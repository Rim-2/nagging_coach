## 2. 아키텍처 다이어그램 (30초)

## 3. 라이브 시연 — 닫힌 루프 4단계 (2분 30초)

핵심 시연 순서 — *perception → reasoning → action → memory* 가 한 흐름으로 도는 걸 보여줌.

### 사전 준비
- PC 위성 가동 중 (백그라운드)
- 폰 위성 가동 중 (DEMO 임계치 1분/2분)
- 텔레그램 봇 채팅창 띄움
- (선택) Railway Deploy Logs 한쪽에 띄워 메시지 도착 시점 보여주기

### Step 1 — 자연 대화로 정보 수집 (30초)
**사용자**: "오늘 발표 자료 좀 만들어야 되는데"

**기대 봇**: 자연스럽게 캐물어 → `register_today_goal_with_steps` 호출해
sub_steps 분해 (예: "주요 메시지 정리", "슬라이드 초안", "리허설").
state 메모에 자동 저장.

**시청자에게 강조**:
> "*어떤 도구가 호출됐는지* 봇이 알리지 않아도, 백엔드는 sub_steps 트리로
> 진척을 추적합니다. 매번 사용자가 다음 단계 안 물어봐도 됩니다."

### Step 2 — PC/폰 감지 → 적시 잔소리 (1분)
- 시연자: **인스타 1분 켜둠** (또는 카톡 2분)
- 1~2분 후 텔레그램에 잔소리 자동 도착
- (가능하면 Railway 로그에 `[App] (원격) 트리거 [능동적 도파민 스크롤] → 잔소리 발송` 보이기)

**시청자에게 강조**:
> "폰을 *연속으로* 1분 봤을 때만 잡습니다. 알림 답장 같은 짧은 사용은
> 무시해요. `UsageEvents` API 로 ACTIVITY_RESUMED/PAUSED 시각을 페어링해서
> *세션 단위*로 측정합니다."

### Step 3 — 톤 적응 + 회복 톤 (30초)
**사용자**: "아 좀 살살 해줘"

**기대 봇**: 자연스럽게 받아줌. EXTRACT 가 `nag_policy_signal: gentle` 잡아
store 에 저장. 그 다음 잔소리부터 압박 빠진 톤.

**시청자에게 강조**:
> "사용자가 톤 명시 안 해도 발화에서 자동 추출. 우울증 환자·번아웃 사용자가
> 진단 모드로 분류되지 않아요. 자기보고만 따릅니다."

### Step 4 — 자기학습 (30초)
**사용자**: "내 패턴 분석해줘"

**기대 봇**: `analyze_my_patterns` 도구 호출 → 최근 활동 로그를 AI 가
종합 분석해서 3~5개 패턴 자연어 반환. 예: "오전엔 sub_step 완료율 ↑,
저녁엔 인스타 빈도 ↑, 산책 메모 있는 날 mood +0.7".

**시청자에게 강조**:
> "여기 데이터는 *자동 누적* — 사용자가 따로 입력 안 합니다.
> mood log 한 줄, 트리거 발사, 목표 완료 같은 게 알아서 쌓여서
> 한 달 분량으로 패턴이 보입니다."

## 5. 자기학습·로드맵 (30초)

> "이미 들어간 거: 자주 잡힌 앱을 *자동으로* 약점 후보로 누적해서 주간 회고 때
> 사용자에게 '이거도 약점이야?' 확인하는 자기학습 메커니즘.
>
> 로드맵: PC·폰 데이터를 시간축에서 합쳐서 'PC 업무앱 + 폰 SNS = 회피' 같은
> 단일 위성으론 못 잡는 패턴을 잡는 *크로스 디바이스 통합 분석*. 백엔드는
> 이미 `POST /trigger` 로 통합되어 있어서 데이터 모델만 더하면 됩니다."

## 6. 마무리 한 문장 (30초)

> 데이터로 격려해줍니다. *능동성 + 친밀감 + 행동과학 기법* — 이 세 축을

## 시연 사고 대비

### 잔소리가 안 올 때
- 클라우드 쿨다운 (10분) 에 걸려 스킵 — Railway 로그에서 `쿨다운 중 (Ns 남음)` 확인
- 같은 trigger_value 라면 **다른 트리거 종류로 전환** (인스타 → 카톡)
- 정 안 되면 발표 흐름 그대로 가고 "방금 발사한 게 클라우드 쿨다운에 걸렸어요 —
  이게 잔소리 폭주 방지 안전장치예요" 로 *기능 차원* framing

### 네트워크 끊김
- Railway 도메인 `/health` 확인
- 폰 위성 알림창에 "감시 중" 안 보이면 앱 재시작
- 가장 안전: 화면 녹화 백업 영상 미리 준비

### 텔레그램 너무 많이 보내는 경우
- 평소 잔소리 미리 다 받아둬서 *데모 직전* 봇이 *조용한 상태* 만들기
- 또는 `/reset` 한 번 (단 다 비워짐 — 데모 데이터 다시 만들어야)
